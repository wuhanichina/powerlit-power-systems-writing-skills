# Letter Precision Reference

Use this reference for IEEE power-system Letters. A Letter should read as if every sentence is under page pressure and every claim is under review pressure.

## Core Standard

Every sentence must either:

- state the narrow problem;
- define the minimal setting;
- introduce a necessary equation or construction;
- derive the key implication;
- verify the claim;
- state the boundary.

Delete everything else.

## One-Claim Discipline

The Letter should not defend a broad project. It should establish one technical point sharply.

Avoid:

- broad motivation after the first page;
- multi-part contribution lists;
- defensive limitation paragraphs;
- secondary experiments;
- full-paper framework language.

Prefer:

- "This letter shows that..."
- "The counterexample demonstrates..."
- "The formulation reduces to..."
- "The numerical check verifies..."

## Reader-Burden Rule

Use this paragraph discipline for Letter prose: 先亮观点，再给理由；一段一意，五句为限；句句支撑，不跑不散。

- Put the paragraph's judgment in the first or second sentence.
- Use the remaining sentences only for the reason, equation, comparison, evidence, or boundary that supports that judgment.
- Split or delete any sentence that opens a second point.
- Deliver the claim clearly, credibly, and with low processing burden.

## Final Cut Pass

Before final output:

1. Mark the one sentence that carries the Letter claim.
2. Remove any paragraph that does not expose, derive, or verify that sentence.
3. Replace defensive language with a direct boundary.
4. Keep only equations referenced by the claim or evidence.
