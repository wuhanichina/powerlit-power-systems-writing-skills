# Letter Introduction Reference

## Goal

The Letter introduction should cut directly to the technical nerve. It does not have room for a full literature landscape.

## Recommended Flow

1. Define the system object or assumption under scrutiny.
2. State why it matters operationally or mathematically.
3. Summarize only the closest prior line.
4. Identify the gap or wrong assumption.
5. Explain the technical reason or missing mechanism.
6. State the Letter contribution in one paragraph.
7. End with a compact structure sentence only if needed.

## Gap Style

A Letter gap should be specific enough to be answered in 4 pages:

- "The common convexity assumption fails under ..."
- "Existing sensitivity calculations require solving ..."
- "Current models omit ..."
- "The RMS model cannot capture ..."
- "The largest RoCoF is not necessarily located at ..."

Avoid broad gaps:

- "Existing studies are insufficient."
- "Few works consider this problem comprehensively."
- "The method improves accuracy and efficiency."

## Contribution Style

Use one central contribution:

- `This letter derives ...`
- `This letter demonstrates ...`
- `This letter proposes ...`
- `This letter provides a counterexample showing ...`
- `The key contribution is an analytical ...`

If listing contributions, keep them concrete:

- deliverable 1: formula/model/counterexample/algorithm;
- deliverable 2: minimal validation/implication.

Do not list three broad contributions. That is usually full-paper posture.

## Prior Work Compression

Use method families, not a survey:

- One sentence for the nearest method family.
- One sentence for its assumption or limitation.
- One sentence for why the Letter's claim is unresolved.

Reference count in the corpus is small. Each citation should help define the exact assumption, method class, or benchmark.

## Self-Reference

Prefer `this letter` in TPWRS/TSG Letter prose when referring to the work. Use it sparingly and shift the grammatical subject to the deliverable after the first mention:

- `This letter derives ...`
- `The resulting expression shows ...`
- `The counterexample demonstrates ...`
- `Numerical results verify ...`

Avoid repeated `we` and repeated `this paper`.
