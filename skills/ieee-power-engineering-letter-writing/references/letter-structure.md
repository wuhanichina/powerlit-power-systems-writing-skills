# Letter Structure Reference

## Corpus Signal

Official rule source checked 2026-06-18: IEEE PES Letter original submissions are limited to 3 formatted pages; revisions are limited to 3.5 pages; letters exceeding those limits are not reviewed. IEEE PES author information also says the accepted letter should not exceed 4 pages after revision and editing.

The PowerLit Letter corpus contained 69 IEEE TPWRS/TSG short papers selected by page count and Letter markers. Treat these as descriptive corpus signals, not as official page limits.

Typical values:

- 4 pages.
- 4-5 major sections.
- 5 abstract sentences.
- 7 introduction paragraphs.
- about 15 tagged equations.
- about 9 references.

## Standard Movement

Use one of these structures.

### Analytical or Formulation Letter

1. `I. INTRODUCTION`
2. `II. MODEL / PROBLEM FORMULATION`
3. `III. DERIVATION / PROPOSED METHOD`
4. `IV. NUMERICAL RESULTS / CASE STUDY`
5. `V. CONCLUSION`

### Counterexample Letter

1. `I. INTRODUCTION`
2. `II. COUNTEREXAMPLE / ILLUSTRATIVE SYSTEM`
3. `III. ANALYSIS / IMPLICATIONS`
4. `IV. CONCLUSION`

### Algorithmic Trick Letter

1. `I. INTRODUCTION`
2. `II. PROBLEM SETTING`
3. `III. EFFICIENT CALCULATION / SOLUTION METHOD`
4. `IV. ILLUSTRATIVE EXAMPLE`
5. `V. CONCLUSION`

## Drafting Budget

Use this as a drafting budget inside the official page constraints:

- Abstract and index terms: 0.2 page.
- Introduction: 0.7-1.0 page.
- Technical core: 1.5-2.0 pages.
- Case/validation: 0.8-1.0 page.
- Conclusion and references: 0.3-0.5 page.

If the technical core cannot fit in about two pages, the claim is too broad for a Letter.

## Section Discipline

- Section II should start the technical object immediately.
- Use subsections sparingly.
- Equations should be numbered only when referenced later.
- Place definitions near first use.
- Keep proofs short; push only essential algebra into the main text.
- One figure or table should carry a decisive message, not decorate the draft.

## Title Discipline

Letter titles should name the exact contribution object:

- `On the ...`
- `A ... Model for ...`
- `Analytical ...`
- `... are Non-Convex`
- `... Cannot Capture ...`
- `Efficient ... Calculation`

Avoid full-paper titles that bundle three modules with "framework", "comprehensive", or "integrated" unless the Letter is actually about that object.
